# YTLA 架构与 AI Feature Composition 设计总结

discussed and generated by GPT-5.3

## 一、系统核心设计

### 分层结构

系统整体采用分层与自相似结构设计：

```
core/
  version A/
    module A/
      controller
      service
      repository
      model
    module B/
features/
  feature A/
    module A/
```

### 核心概念

#### Module

- 基础功能单元
- 具有独立 instance
- 拥有自己的数据空间
- 由自身负责数据管理

#### Card

- 全局共享能力
- 不隶属于 plan
- 使用统一 API
- 支持 event 触发机制

#### Plan

- 配置层
- 仅管理 module 数量与类型
- 不管理 module 内部数据

#### Instance 机制

每个 module：

- 拥有独立 instance\_id
- 对应独立数据空间
- 实现数据隔离

#### Caller 机制

**作用**：

- 跨 instance 数据访问
- 执行 CRUD 操作

**特点**：

- 属于 module 定义的一部分
- 同类型 module 共享 caller 能力
- 用于 module 间交互

#### Event 系统（Card）

- Card 使用统一 API
- Event 作为参数传入
- 由具体 card module 判断是否处理

#### 状态与控制

Module 和 Card 支持：

- **status**（运行状态展示）
- **active\_flag**（启用 / 停用）

停用效果：

- Module 不可进入
- Card 不可见

## 二、架构特性

### 自相似结构（Self-similar Architecture）

系统基于：

- `baseModule` → `module`
- `baseCard` → `card`

支持：

- 任意层级嵌套
- 统一 contract
- 结构递归扩展

### 数据隔离优先

**设计原则**：

- 默认隔离
- 必要时通过 caller 共享

### Runtime 级模块系统

区别于传统模块化：

- **传统模块**：功能块、编译期结构、静态调用
- **当前系统**：可运行节点、运行时结构、动态调用（caller）

### Registry 机制

**后端**：

- 使用 Flask Blueprint 扫描加载

**前端**：

- 扫描 registries 目录
- 使用 Vue3 defineAsyncComponent

**示例**：

```javascript
createModuleRegistry('dictionary', config)
createModuleFlowRegistry('dictionary', flowManager)
```

## 三、当前能力边界

### 已具备能力：

1. 模块隔离（instance）
2. 跨模块调用（caller）
3. 统一结构定义（baseModule / baseCard）
4. 动态加载（registry）
5. 部分组件合法存在（结构层）

### 当前缺失能力：

1. 语义层契约（Semantic Contract）
   - **问题**：结构可连接 ≠ 语义可连接
   - **需要定义**：
     - 输入（consumes）
     - 输出（produces）
2. 能力抽象（Capability Layer）
   - **当前**：`caller` = 数据访问
   - **未来**：`caller` = 能力接口
   - **例如**：
     - `getWords()`
     - `addToNotebook()`
3. 关系显式化（Graph）
   - **当前**：隐式调用（caller）
   - **未来**：显式依赖图（module graph）

3. 自动生成模块结构
4. 保留接口

- 或 Feature Composition Runtime

## 六、核心能力拆解

### 部分完成（Partial Feature）

系统允许：

- 不完整但合法的模块存在

**前提**：

- 满足 base contract

### 可组合结构（Composable Graph）

**目标**：

`feature` = `module graph`

**支持**：

- 动态扩展
- 持续拼接
- 多轮演化

### 能力递归

由于结构统一：

`module` → `submodule` → ...

**可以实现**：

能力组合 → 再组合

## 七、关键技术挑战

### 接口留白（Interface Gap）

AI 需要：

- 不生成完整系统
- 只生成可扩展部分

### 语义对齐

**问题**：

`module A` 输出 ≠ `module B` 输入

**解决方向**：

- 语义契约
- 数据 schema

### 拼接一致性

**风险**：

- 数据结构冲突
- 状态流错乱
- 调用不一致
