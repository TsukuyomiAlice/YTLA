# YTLA - Docs 文件规范

## 通过使用harness engineering进行应用内docs的维护

将本文档添加到对话中.  
或者在对话中，根据实际需求，使用不同的提示词.

- 维护应用文档时使用的提示词参考

```
你正在维护这个应用的文档.  
用户应该告诉了你需要维护文档的应用目录. 
这个应用的文档目录位于docs / documents 文件夹下. 
另外，还有一份在应用根目录下的readme.md文档

你需要做的是，结合应用内存在的文件，更新以下范围的文档：
根目录的readme.md
文档目录内的：readme下的.md文档

readme用于介绍应用的**概念**和**文件结构**.
**概念** 里简明描述这个应用用来做什么. 
**文件结构** 需要列出应用下的文件列表. 

一般而言，如果用户没有提出要求，你需要维护的是根目录的readme.md，以及readme目录下的多语言版本文件.  
你要根据根目录的readme.md的语言，确定文档的主语言，并生成其它文档的翻译版本.  
你要先完整地查看应用内的代码文件，确定并替换生成readme.md中需要生成或更新的内容.  
随后，同步其它的翻译版本.  

并不建议你参考其它的应用或功能下的readme.md来为目标应用更新.  
实际上，readme.md是有预生成的统一格式的. 
你只需要完成占位符的填充即可.  

**作者名** 有一个特别约定:
一般不要主动改动. 
但是，如果出现了"Official"，则表明这个应用是官方应用. 你要同步更新所有的作者名为"Official"
如果用户修改了其它的作者名，那么做同步的修改，把所有的readme文档中的作者名改成相同的名字. 

如果发生了更新的话，顺带更新一下文档里的**文件更新日期**.格式为"YYYY-MM-DD".
但是，如果没有更新任何内容，那么不要更新*文件更新日期**.

对于文档目录内其它目录下的文档，除非用户明确要求，一般不主动维护.   

有时候，用户可能会在其它语言目录中更新了内容，而没有更新主目录的readme.md.  
这时候你需要进行比对，确认哪一个版本才是最具体的版本，并以那个版本所使用的语言作为主语言，更新并同步所有的readme.md. 
```

## 总述

本文件规范适用于以下范围.  
文件规范提供的是一个流程上的指导，在实际操作中以用户的指定需求优先.

适用核心名: 全部  
适用栈: 包括前后端  
适用语言类型和框架: 全部

指定规范目录：docs  
指定规范文档：readme.md

## 标准文件结构框架

### 文档目录可出现在以下层级目录中：

- YTLA根目录
    - docs 描述YTLA
    - YTLA前端目录
        - docs 描述YTLA前端
        - core
            - core_版本名
                - docs 描述版本
                - core特性
                    - docs 描述特性
                    - 具体特性
                        - docs 描述具体特性
        - features
            - feature_功能名
                - docs 描述功能
                - feature_功能形式
                    - feature_具体类型
                        - docs 描述具体类型
    - YTLA后端目录
        - docs 描述YTLA后端
        - core
            - core_版本名
                - docs 描述版本
                - core特性
                    - docs 描述特性
                    - 具体特性
                        - docs 描述具体特性
        - features
            - feature_功能名
                - docs 描述功能
                - feature_功能形式
                    - feature_具体类型
                        - docs 描述具体类型

### 文档目录的泛用格式如下：

- 原目录
    - docs 文档目录
        - readme readme目录，必须包含
            - en-US 英语
            - zh-CN 简体中文
            - 其它语种
        - 其它文档
    - readme.md 根目录下需有readme.md

#### 目前仅readme有强制文件目录的格式要求.

#### docs文档目录名称可能随语言框架设计而发生改变，而其内部保持不变.

- 比如，在python-flask框架下，docs文档的目录名称为docs,  
  而在vue3框架下，docs文档的目录名称则为documents.

#### 理论上，其它文档的目录应遵循每一种文档类型，均有一个文件夹用于单独存放该类文档.

### 其它类型的文档目录

这里会添加其它的，已经出现在docs目录下的其它目录名称.  
对于新的目录名称，将会在这里列举并阐述此名称下文档的用意.

| 目录名        | 用途                                  |
|------------|-------------------------------------|
| readme     | 文档readme目录                          |
| design     | 介绍想法设计, 功能 story, 用户 scenario 等     |
| rules      | 规范                                  |
| implements | 针对单一文档的技术说明                         |
| tech       | 针对模组/整个feature层面的技术说明               |
| discuss    | 设计讨论, 进度汇总, todo清单 等文档              |
| interfaces | 接口设计文档，比如openAPI的yaml文档，或者接口定义md文档等 |

## docs文档的书写规则

在脚手架中可以预生成readme相关目录和文档，并且会生成固定格式的表单.  
优先更新位于根目录下的readme.md.  
之后，可以在使用ai工具的过程中，要求ai同步完成docs/readme目录下的各个语种文件的更新.  
如果用ai工具生成了整个项目，那么需要要求ai先更新根目录的readme.md文档，然后同步更新docs/readme目录下的各语种对应的readme.md.
