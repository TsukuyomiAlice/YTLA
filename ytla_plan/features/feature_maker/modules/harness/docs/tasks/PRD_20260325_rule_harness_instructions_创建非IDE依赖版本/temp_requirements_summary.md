# rule_harness_instructions_ver_direct.md 需求汇总

## 一、文档变更概览

本文档汇总了从 `rule_harness_instructions.md` 到 `rule_harness_instructions_ver_direct.md` 的所有变更内容。

## 二、新增章节

### 2.1 1.3 工作环境假设（新增）

**位置**：第 1 章总述，在 1.2 设计理念之后

**内容概要**：
- 说明文档适用于任意大模型对话环境，无需特定IDE
- 文件操作说明：读取、写入、修改、列出目录的具体方式
- 用户交互说明：通过自然语言对话进行
- 子代理协作说明：通过分段对话或独立对话实现

## 三、移除的IDE特定内容

### 3.1 移除的命令引用

| 原内容 | 替换为 | 位置 |
|--------|--------|------|
| `/spec` 命令 | 用户明确要求进行项目开发 | 0.2, 2.1 |
| `/plan` 命令 | 用户明确要求进行规划 | 多处 |

### 3.2 移除的工具引用

| 原工具 | 替换为 | 说明 |
|--------|--------|------|
| `TodoWrite` | 自然语言对话告知用户 | 任务管理 |
| `Read` | 请用户提供文件内容或描述 | 读取文件 |
| `Write` | 提供完整文件内容供用户保存 | 写入文件 |
| `Edit` / `SearchReplace` | 提供修改内容或完整文件 | 修改文件 |
| `LS` / `Glob` / `Grep` | 请用户提供目录结构或搜索结果 | 代码库探索 |
| `AskUserQuestion` | 自然语言对话提问 | 用户交互 |
| `NotifyUser` | 自然语言对话通知 | 通知用户 |
| `subagent` / `general_purpose_task` / `search` | 分段对话或独立对话 | 子代理协作 |

## 四、调整的工作流程

### 4.1 Start Stage 调整

**原流程**：
- 探索代码库（使用 IDE 工具）
- 检索已有 PRD（使用 IDE 工具）
- 创建 PRD 目录（使用 IDE 工具）

**新流程**：
- 请用户提供当前代码库状态
- 请用户检查目标模组的 docs/tasks 区域
- 请用户在指定位置创建目录

### 4.2 Spec Mode 调整

**原流程**：
- 使用 `NotifyUser` 通知用户审查

**新流程**：
- 请用户审查规划文档（自然语言）

### 4.3 Agent Mode 调整

**原流程**：
- 使用 `TodoWrite` 标记任务状态
- 使用子代理工具委派任务

**新流程**：
- 告知用户正在处理该任务
- 建议通过分段对话处理复杂任务
- 告知用户任务完成情况

## 五、actions.md 规范调整
